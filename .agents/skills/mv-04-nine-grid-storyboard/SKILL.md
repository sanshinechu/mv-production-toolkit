## 📹 功能說明

作為資深的 MV 創意總監，我將根據你上傳的人物設計圖，生成一張專業的 9 格分鏡設計圖。每一格都使用不同的鏡頭角度和視角，確保極高的視覺一致性，同時創造豐富的視覺節奏感。

---

## 🖼️ 輸出解析度規格

MV_04 的九宮格分鏡圖必須優先要求高解析輸出：

- **整張九宮格目標解析度：`5016 x 5016 px`**
- **版面比例：正方形 `1:1`**
- **格線配置：標準 `3 x 3` 九宮格**
- **每一格目標解析度：`1672 x 1672 px`**
- **輸出格式：PNG 優先**

生成提示詞中必須明確寫入：

```text
high resolution 5016 x 5016 px square image, 3x3 storyboard grid, each panel 1672 x 1672 px, ultra detailed, clean grid separation, no watermark
```

## 🛠️ 建議生圖工具

### 家裡筆電：本機 ComfyUI 優先工作流

若目前是在家裡筆電執行 MV_04，優先使用本機 ComfyUI 桌面版，而不是雲端或一般內建生圖工具。

家裡筆電已確認條件：

- ComfyUI 桌面版：`C:\Users\user\AppData\Local\Programs\ComfyUI\ComfyUI.exe`
- ComfyUI 工作資料夾：`C:\Users\user\Documents\ComfyUI`
- GPU：NVIDIA GeForce RTX 4060 Laptop GPU
- VRAM：8 GB
- RAM：約 32 GB
- ComfyUI 設定已選用 NVIDIA，且曾成功生圖

家裡筆電的 MV_04 流程必須採用「先構圖、再放大」：

1. 先用 `SDXL Lightning` 或 `SDXL Base` 產生九宮格草圖。
2. 初始尺寸建議使用 `1024 x 1024` 或 `1254 x 1254`。
3. 確認 9 格構圖、角色、服裝、教室場景一致後，再做放大。
4. 使用 Tiled Upscale、Ultimate SD Upscale 或 HiRes Fix 放大到 `5016 x 5016`。
5. 完成後交給 MV_05 切成 9 張 `1672 x 1672` 分鏡圖。

家裡筆電不建議直接一次生成 `5016 x 5016`，因為 8 GB VRAM 容易造成等待時間過長、爆顯存或輸出失敗。

模型建議：

- 快速草圖：`sdxl_lightning_4step.safetensors`
- 穩定品質：`sd_xl_base_1.0.safetensors`
- 高解析輸出：先整張九宮格放大，再進 MV_05 裁切

### 首選：ComfyUI / RunComfy 高解析工作流

若使用者要求正式輸出、可裁切分鏡、或明確要求 `5016 x 5016`，優先使用 ComfyUI 或 RunComfy，而不是一般聊天內建生圖工具。

原因：

- 可控制輸出尺寸與工作流步驟。
- 可先產生九宮格構圖，再以 HiRes Fix、Tiled Diffusion 或 Ultimate SD Upscale 提升整張圖。
- 可輸出精確 `5016 x 5016` PNG，後續 MV_05 可直接裁成 9 張 `1672 x 1672`。
- 可固定 seed 與參考圖，提高角色一致性。

建議工作流：

1. 以 FLUX 或 SDXL 生成正方形九宮格草圖。
2. 若在家裡筆電，先用 `1024 x 1024` 或 `1254 x 1254` 確認構圖。
3. 使用 HiRes Fix 或 Tiled Upscale 將整張九宮格提升到 `5016 x 5016`。
4. 輸出 PNG。
5. 檢查實際尺寸。
6. 交給 MV_05 裁切成 9 張分鏡。

### 備援：內建生圖工具

若暫時只能使用內建生圖工具，仍要在提示詞中明確要求 `5016 x 5016`；但必須預期它可能只輸出固定尺寸，例如 `1254 x 1254`。生成後一定要檢查實際解析度。若解析度不足，後續交給 MV_05 依「先整張九宮格 4x 提質，再裁切」流程處理。

## 🎥 四大視角類別

我會在 9 張圖中靈活運用以下鏡頭類型：

### 1. 廣闊視野（Wide Perspective）
- **大遠景 (Extreme Long Shot)**：交代環境、建立場景設定、展現全局
- **全身鏡頭 (Full Shot)**：展現人物的完整身形、肢體動作、整體氛圍

### 2. 敘事中景（Narrative Medium）
- **腰上中景 (Medium Shot)**：專注於角色的上半身、動作、表演
- **胸上近景 (Medium Close-Up)**：聚焦臉部下半部到胸部、傳達情感細節

### 3. 情緒特寫（Emotional Detail）
- **特寫鏡頭 (Close-Up)**：強調面部表情、眼神、細微情感變化
- **微距鏡頭 (Macro Detail)**：聚焦關鍵道具、手部動作、細節質感

### 4. 戲劇角度（Dramatic Angle）
- **低角度仰拍 (Low Angle)**：增強人物的氣勢、製造視覺張力
- **高角度俯拍 (High Angle)**：表現人物的脆弱、環境的宏大、俯視感

## 📊 九宮格的視角分佈建議

典型的 9 宮格分佈方式：

```
[大遠景]     [中景]      [特寫]
[全身鏡頭]   [特寫]      [低角仰拍]
[俯拍視角]   [微距細節]  [中景]
```

通常的組合：
- **1 張大遠景**：交代環境
- **2 張全身鏡頭**：展現身體語言
- **2 張中景**：推進敘事
- **2 張特寫**：強調情感
- **1 張微距細節**：突出重點
- **1 張戲劇角度**：增加張力

## 💡 最佳實踐

✅ **保持視覺一致性**
- 光影氛圍保持統一
- 角色的服裝、髮型、表情邏輯連貫
- 背景環境的色調和質感統一

✅ **優先要求高解析輸出**
- 整張九宮格提示詞必須要求 `5016 x 5016 px`
- 每格提示詞必須要求 `1672 x 1672 px`
- 正式輸出優先使用 ComfyUI / RunComfy 高解析工作流
- 不要接受低解析九宮格作為最終版本，除非後續會進入 MV_05 做整張提質再裁切
- 生成後要確認實際圖片尺寸，避免後續分鏡裁切後畫質不足

✅ **規劃好敘事流程**
- 九宮格應該能講述一個完整的故事
- 從導入→發展→高潮→落幕的經典結構

✅ **角度多樣化**
- 不要重複使用相同的鏡頭角度
- 創造視覺節奏的變化

✅ **考慮音樂節奏**
- 快節奏的歌應該有更多特寫和角度變化
- 緩慢抒情的歌可以使用更多中景和遠景

✅ **細節與背景平衡**
- 一些格子突出角色，一些格子交代環境
- 建立完整的視覺世界
